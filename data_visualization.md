# 2. 데이터 시각화 규칙

작성자: 김다은

## 0. 개념 정리

### 분석 대상 문항
- 시각화와 해석의 중심이 되는 문항입니다.
- 차트, 인사이트, 데이터 테이블은 모두 분석 대상 문항의 응답 결과를 기준으로 구성합니다.
- 문항명은 분석 대상 문항의 `question_label`을 사용합니다.
- 질문 원문은 분석 대상 문항의 `question_full`을 사용합니다.

### 분석 기준 문항
- 분석 대상 문항의 결과를 그룹별로 나누어 비교할 때 기준이 되는 문항입니다.
- 분석 기준 문항은 기본적으로 `객관식 단일` 문항만 사용할 수 있습니다.
- 분석 기준 문항은 문항 코드북에서 `response_type = 객관식 단일`인 문항만 사용할 수 있습니다.
- 분석 기준 문항이 적용되지 않은 경우, 분석 대상 문항의 전체 결과만 표시합니다.
- 분석 기준 문항이 적용된 경우, 분석 대상 문항의 결과를 기준 문항의 응답값별로 나누어 비교합니다.

## 1. 공통 구성 요소

레이아웃 가안 이미지는 별도 참고합니다.

### 수치 표시 및 반올림 규칙
- 비율(%)은 계산에 사용한 원래 값으로 산출한 뒤, 표시 시점에만 소수점 첫째 자리까지 반올림합니다.
- 소수점 둘째 자리 값이 `5` 이상이면 올림하는 `round half up` 방식을 사용합니다.
  - 예: `48.55% → 48.6%`, `32.64% → 32.6%`
- 차트 데이터 레이블, 툴팁, 데이터 테이블 등 비율을 노출하는 모든 UI는 동일한 반올림 규칙을 사용합니다.
- 응답자 수 `N`은 반올림하지 않고 원래 정수값 그대로 표시합니다.

### 응답 보기 정렬 및 노출 규칙
- 객관식 문항의 응답 보기 순서는 기본적으로 코드북의 `response_options` 순서를 따릅니다.
- 코드북에 정의된 보기가 먼저 나오고, 데이터에만 존재하는 예외 값이 있을 경우에만 코드북 정의 뒤에 추가합니다.
- 응답자 수가 `0`인 보기는 차트, 툴팁, 데이터 테이블에서 모두 표시하지 않습니다.
- 그룹별 비교에서도 전체 응답 기준으로 `0건`인 보기는 동일하게 숨깁니다.
- 보기 순서를 숨겨진 항목 때문에 재정렬하지 않으며, `0건이 아닌 보기만 남긴 원래 순서`를 유지합니다.

### 현재 HTML 기준 그래픽 규칙
- 객관식 단일의 기본 차트는 `가로 막대 + 우측 수치 레이블` 구조를 사용합니다.
- 각 행은 `응답 보기 라벨 / 막대 영역 / 비율 값`의 3열 구조로 배치합니다.
- 응답 보기 라벨은 한 줄 말줄임 처리하고, 값 영역은 `tabular-nums`로 표시해 숫자 자릿수 정렬을 안정적으로 유지합니다.
- 막대 트랙은 연한 회색 배경의 둥근 직사각형이며, 실제 값 막대는 그 위를 채우는 방식으로 표시합니다.
- 기본 막대는 단일 색상 `#5a6674`를 사용합니다.
- 그룹 비교 차트의 전체 응답 막대는 연한 회색 `#e4e4e4`를 사용하고, 그룹별 결과는 색상 점과 선을 중첩하여 표시합니다.
- 그룹 비교 선은 얇은 라운드 꺾은선으로 그리고, 각 보기 위치에 원형 점 마커를 함께 표시합니다.
- 그룹별 색상은 현재 다음 팔레트를 순환 사용합니다.
  - `#5b7a9a`, `#c67b7b`, `#7ba87a`, `#c6a77b`, `#a77bc6`, `#7bbfb8`, `#c67bad`, `#9ba07b`
- 범례는 체크박스와 색상 스와치를 함께 제공하며, 사용자가 특정 그룹의 표시 여부를 켜고 끌 수 있어야 합니다.
- 숨김 처리된 그룹은 범례 텍스트를 약한 색으로 바꾸고, 스와치 불투명도도 낮춰 비활성 상태를 시각적으로 구분합니다.
- 데이터 테이블은 차트 바로 아래에서 동일한 보기 순서를 유지하며, 헤더는 연한 회색 배경으로 구분합니다.
- 그룹 비교 테이블은 `응답자 전체`와 각 그룹을 나란히 배치하고, 그룹 열 시작 지점에 세로 구분선을 둡니다.
- 합계 행은 본문보다 강조된 배경과 굵기로 표시합니다.
- 툴팁은 짙은 배경의 작은 플로팅 패널로 표시하며, 보기명/비율/응답자 수(N)처럼 차트 해석에 필요한 최소 정보만 제공합니다.

### 문항명
- `분석 대상 문항`의 `question_label`을 표시하는 영역입니다.
- 사용자는 이 값을 통해 현재 어떤 문항의 결과를 보고 있는지 확인합니다.

### 질문 원문
- `분석 대상 문항`의 `question_full`을 표시하는 영역입니다.
- 문항명만으로 의미가 충분히 전달되지 않을 때, 질문의 전체 맥락을 이해할 수 있도록 제공합니다.

### 차트
- `분석 대상 문항`의 응답 결과를 시각화하는 핵심 영역입니다.
- 필요 시 `분석 기준 문항`에 따라 그룹별 비교 결과가 함께 표시될 수 있습니다.

### 툴팁
- 차트 요소(막대, 포인트, 선 등)에 마우스를 올렸을 때 표시되는 상세 정보입니다.
- 해당 항목의 값, 비율, 응답자 수, 그룹명 등 차트를 읽는 데 필요한 정보를 제공합니다.
- 그룹명은 `분석 기준 문항명: 그룹 값` 형식으로 표기합니다.
  - 예: `성별: 여`, `세대: Z세대`

### 범례
- 차트에 사용된 색상 또는 표시 요소가 무엇을 의미하는지 설명하는 영역입니다.
- 그룹별 비교가 있는 경우에만 표시합니다.

### 인사이트
- 차트와 데이터에서 확인되는 주요 경향이나 특징을 AI가 요약해 설명하는 영역입니다.
- 통계적 검정 결과는 아니며, 사용자의 해석을 돕기 위한 참고용 설명입니다.

### 데이터 테이블
- 차트에 사용된 수치 데이터를 표 형태로 보여주는 영역입니다.
- 사용자는 이 영역에서 시각화에 사용된 실제 값을 직접 확인할 수 있습니다.
- 각 응답 보기별 선택 비율(%)과 선택 응답자 수(N)를 함께 표시합니다.
- 그룹별 비교 차트에서는 전체 결과와 그룹별 결과를 함께 제공합니다.

### 기타 응답 처리 규칙
- 문항 코드북에서 `other_input_expected = Y`인 문항은 `기타`를 하나의 응답 보기로 집계합니다.
- 직접입력 텍스트는 화면에 고정 노출하지 않으며, `상세 응답 확인` 버튼을 통해 별도 창에서 제공합니다.
- 인사이트에서는 필요 시 기타 응답의 주요 내용을 요약하여 참고용으로 제시할 수 있습니다.

## 2. 응답 형식별 시각화 규칙

### 2-1. 객관식 단일

집계 기준:
- 응답 데이터셋에서 해당 문항의 `raw` 1개 열을 사용하여 보기별 비율을 집계합니다.
- 응답 보기 순서는 코드북의 `response_options` 순서를 우선 사용합니다.
- 코드북에는 있으나 실제 응답자 수가 `0`인 보기는 표시하지 않습니다.
- 데이터에만 존재하는 예외 값이 있을 경우에는 코드북 정의 보기를 모두 배치한 뒤, 뒤쪽에 추가합니다.

#### 기본 차트
- 기본 차트는 가로 막대 그래프로 표시합니다.
- 가로축: 응답 비율 `(0~100%)`
- 세로축: 응답 보기
- 데이터 레이블: 응답 비율 `%`
- 데이터 레이블 위치: 막대가 끝나는 지점의 바깥쪽 옆

#### 그룹별 비교 차트
- 전체 응답 결과는 연한 회색의 가로 막대로 표시합니다.
- 그룹별 응답 결과는 그 위에 꺾은선으로 중첩하여 표시합니다.
- 가로축: 응답 비율 `(0~100%)`
- 세로축: 응답 보기
- 전체 결과: 연한 회색 가로 막대
- 그룹별 결과: 그룹별 색상을 적용한 꺾은선
- 데이터 레이블: 표시하지 않음
- 그룹별 비교 차트에서도 응답자 전체 기준 `0건`인 보기는 표시하지 않습니다.

#### 툴팁
- 기본 차트 막대:
  - 1행: 응답 보기 이름
  - 2행: 응답 비율 `xx.x%`
  - 3행: 응답자 수 `N=xxx`
- 그룹 비교 차트의 연한 회색 막대:
  - 1행: 응답자 전체
  - 2행: 응답 보기 이름
  - 3행: 응답 비율 `xx.x%`
  - 4행: 응답자 수 `N=xxx`
- 그룹 비교 차트의 꺾은선 점:
  - 1행: 그룹명
  - 2행: 응답 보기 이름
  - 3행: 응답 비율 `xx.x%`
  - 4행: 응답자 수 `N=xxx`

#### 범례
- 기본 차트: 표시하지 않습니다.
- 그룹 비교 차트:
  - 그룹별 색상과 이름을 표시합니다.
  - 그룹별 체크박스를 제공하여, 사용자가 특정 그룹의 표시 여부를 제어할 수 있도록 합니다.

### 2-2. 객관식 중복

집계 기준:
- 응답 데이터셋에서 해당 문항에 연결된 `expanded` 열들을 함께 사용하여 보기별 선택 비율을 집계합니다.

#### 기본 차트
- 기본 차트는 가로 막대 그래프로 표시합니다.
- 객관식 중복은 1명의 응답자가 여러 보기를 선택할 수 있으므로, 각 보기별 응답 비율이 아니라 `선택 비율(Y/N)`을 기준으로 집계합니다.
- 가로축: 응답 비율 `(0~100%)`
- 세로축: 응답 보기
- 데이터 레이블: 선택 비율 `%`
- 데이터 레이블 위치: 막대가 끝나는 지점의 바깥쪽 옆

#### 그룹별 비교 차트

- 그룹별 응답 결과는 그 위에 꺾은선으로 중첩하여 표시합니다.
- 가로축: 선택 비율 `(0~100%)`
- 세로축: 응답 보기
- 전체 결과: 연한 회색 가로 막대
- 그룹별 결과: 그룹별 색상을 적용한 꺾은선
- 데이터 레이블: 표시하지 않음

#### 툴팁
- 기본 차트 막대:
  - 1행: 응답 보기 이름
  - 2행: 응답 비율 `xx.x%`
  - 3행: 응답자 수 `N=xxx`
- 그룹 비교 차트의 연한 회색 막대:
  - 1행: 응답자 전체
  - 2행: 응답 보기 이름
  - 3행: 응답 비율 `xx.x%`
  - 4행: 응답자 수 `N=xxx`
- 그룹 비교 차트의 꺾은선 점:
  - 1행: 그룹명
  - 2행: 응답 보기 이름
  - 3행: 응답 비율 `xx.x%`
  - 4행: 응답자 수 `N=xxx`

#### 범례
- 기본 차트: 표시하지 않습니다.
- 그룹 비교 차트:
  - 그룹별 색상과 이름을 표시합니다.

### 2-3. 객관식 순위

집계 기준:
- 응답 데이터셋에서 해당 문항에 연결된 `expanded` 열들을 함께 사용하여 `1순위`, `2순위`, `3순위` 등 각 순위에 대하여 보기별 응답 비율을 따로 집계합니다.
- 차트 상단의 순위 요약은 순위별 응답에 가중치를 적용한 종합 점수를 기준으로 결정합니다.
  - 예: 4순위까지 있는 경우 `1순위 = 4점 | 2순위 = 3점 | 3순위 = 2점 | 4순위 = 1점`

#### 기본 차트
- 차트 상단에는 각 보기의 종합 순위를 텍스트로 표시합니다.
- 동률이 발생한 경우에는 공동 순위로 표시할 수 있습니다.
- 차트 하단에는 보기별 순위 분포를 보여주는 누적 가로 막대 그래프를 표시합니다.
- 가로축: 순위별 응답 비율의 합 `(0~100%)`
- 세로축: 응답 보기
- 누적 구간: `1순위`, `2순위`, `3순위` 등 각 순위를 색상으로 구분하여 표시
- 데이터 레이블: 누적 응답 비율 `%`
- 데이터 레이블 위치: 막대가 끝나는 지점의 바깥쪽 옆

#### 그룹별 비교 차트
- 기본 누적 막대 그래프를 표시하지 않습니다.
- 대신 그룹별 순위 결과를 텍스트로 제공합니다.
- 각 그룹별로 종합 순위를 한 줄 또는 여러 줄의 텍스트로 표시합니다.
  - 예시:
    - `성별: 남성 → 1위 A > 2위 B > 3위 C > 4위 D`
    - `성별: 여성 → 1위 B > 2위 A > 3위 D > 4위 C`
- 그룹별 종합 순위도 전체 결과와 동일하게, 순위별 응답에 가중치를 적용한 종합 점수를 기준으로 계산합니다.
- 그룹별 체크박스를 제공하여, 사용자가 특정 그룹의 표시 여부를 제어할 수 있도록 합니다.

#### 툴팁
- 기본 차트 누적 막대의 각 순위 구간:
  - 1행: 응답 보기 이름
  - 2행: 순위
  - 3행: 응답 비율 `xx.x%`
  - 4행: 응답자 수 `N=xxx`
- 그룹별 순위 요약 텍스트:
  - 1행: 그룹명
  - 2행: 응답 보기 이름
  - 3행부터: 각 순위별 응답 비율 `xx.x%`
  - 마지막 행: 응답자 수 `N=xxx`

#### 범례
- 기본 차트:

### 2-4. 객관식 척도

#### 2-4-1. 객관식 척도_raw(원문항)

집계 기준:
- 응답 데이터셋에서 해당 문항의 `raw` 1개 열을 사용합니다.
- 각 응답값을 척도 점수별로 집계하여, `1점`, `2점`, `3점` 등 각 점수에 대한 응답 비율을 계산합니다.
- 평균값은 해당 문항의 유효 응답값을 기준으로 산술평균으로 계산합니다.
- 척도 길이는 문항 코드북의 `value_count`를 기준으로 판단합니다.
- 점수 라벨은 문항 코드북의 `value_code_map`을 기준으로 사용합니다.

#### 기본 차트
- 점수별 응답 분포를 보여주는 `100% 누적 가로 막대그래프`
- 가로축: 점수별 응답 비율의 합 `(0~100%)`
- 세로축: 그룹명
  - 기본 차트의 경우 `응답자 전체` 1개 행만 표시
- 누적 구간: `1점`, `2점`, `3점` 등 각 점수를 서로 다른 색상으로 구분
- 데이터 레이블: 각 누적 구간별 응답 비율 `%`
- 데이터 레이블 위치: 각 누적 구간의 중앙
  - 구간 폭이 좁아 중앙 표기가 어려운 경우에는 주변 여백에 표시 가능
- 평균값은 누적 막대그래프 상단에 별도로 표시
- 평균값 위치를 표시하는 영역은 누적 막대그래프의 가로축과 같은 길이로 구성
- 축선은 표시하지 않고, 시작점과 끝점을 비롯한 각 점수의 눈금만 표시
- 가장 왼쪽 값은 항상 `1`, 가장 오른쪽 값은 해당 척도의 최고점
- 평균값의 정확한 위치에 `▼` 표시를 두고, 그 위에 평균값 숫자를 표기

#### 그룹별 비교 차트
- 각 그룹의 점수별 응답 분포를 `100% 누적 가로 막대그래프`를 세로로 나열하여 표시
- 가로축: 점수별 응답 비율의 합 `(0~100%)`
- 세로축: 그룹명
- 누적 구간: `1점`, `2점`, `3점` 등 각 점수를 서로 다른 색상으로 구분
- 데이터 레이블: 각 누적 구간별 응답 비율 `%`
- 데이터 레이블 위치: 각 누적 구간의 중앙
  - 구간 폭이 좁아 중앙 표기가 어려운 경우에는 주변 여백에 표시 가능
- 각 그룹의 평균값은 해당 누적 막대그래프 상단에 별도로 표시
- 평균값 위치를 표시하는 영역은 각 누적 막대그래프의 가로축과 같은 길이로 구성
- 축선은 표시하지 않고, 시작점과 끝점을 비롯한 각 점수의 눈금만 표시
- 가장 왼쪽 값은 항상 `1`, 가장 오른쪽 값은 해당 척도의 최고점
- 각 그룹 평균값의 정확한 위치에 `▼` 표시를 두고, 그 위에 평균값 숫자를 표기
- 각 그룹명의 왼쪽에는 체크박스를 제공하여, 사용자가 특정 그룹의 표시 여부를 제어할 수 있도록 합니다.

#### 툴팁
- 기본 차트의 누적 막대:
  - 1행: 응답 점수 `(1점, 2점 등)`
  - 2행: 응답 비율 `xx.x%`
  - 4행: 응답자 수 `N=xxx`
- 기본 차트의 평균 표식:
  - 1행: 평균 `xx.x점`
  - 2행: 척도 범위 `(5점 척도, 7점 척도 등)`
- 그룹 비교 차트의 누적 막대:
  - 1행: 그룹명
  - 2행: 응답 점수 `(1점, 2점 등)`
  - 3행: 응답 비율 `xx.x%`
  - 4행: 응답자 수 `N=xxx`
- 그룹 비교 차트의 평균 표식:
  - 1행: 그룹명
  - 2행: 평균 `xx.x점`
  - 3행: 척도 범위 `(5점 척도, 7점 척도 등)`

#### 범례
- 점수별 색상을 설명하는 용도로 표시합니다.
- `1점`, `2점`, `3점` 등 각 점수에 대응하는 색상을 표시합니다.
- 필요 시 점수 라벨을 함께 표시할 수 있습니다.
  - 예: `1점(전혀 그렇지 않다)`, `5점(매우 그렇다)`

#### 중간값 숨김 버튼
- 객관식 척도 raw 문항에서는 중간값 숨김 옵션을 제공합니다.
- 이 옵션은 시각적으로만 중간값 구간을 숨기는 기능이며, 응답 비율, 평균값, 응답자 수 등의 집계 기준은 변경하지 않습니다.
- 중간값은 홀수 척도에서만 발생합니다.
  - 예: 5점 척도에서는 `3점`
  - 예: 7점 척도에서는 `4점`

#### 2-4-2. 객관식 척도_derived(산출 지표)

집계 기준:
- 응답 데이터셋에서 해당 문항의 `derived` 1개 열을 사용합니다.
- 각 셀 값은 문항 코드북의 `formula` 열에 정의된 계산식을 따라 산출된 숫자이며, 소수점을 포함한 연속형 수치입니다.
- 평균값은 해당 문항의 유효 응답값을 기준으로 산술평균으로 계산합니다.

축 범위 규칙:
- 축 범위는 문항 코드북의 `value_count` 자체가 아니라, `formula`에 따라 산술적으로 가능한 최소값과 최대값을 기준으로 설정합니다.
- 원 척도 평균값인 경우에는 원 척도의 `value_count`를 고려하여 최소값~최대값을 사용합니다.
- 합산 점수인 경우에는 문항 수와 각 문항의 최소값·최대값을 반영한 총합 범위를 사용합니다.
- 환산 점수인 경우에는 해당 환산 규칙에 따른 이론적 범위를 사용합니다.
- 산술적 범위를 명확히 계산하기 어려운 경우에는 실제 관측값의 최소값~최대값을 사용하되, 보기 쉬운 단위로 반올림하여 표시할 수 있습니다.

- 세로축: 그룹명
  - 기본 차트의 경우 `응답자 전체` 1개 행만 표시
- 평균값은 바이올린 차트 상단에 별도로 표시
- 평균값 위치를 표시하는 영역은 바이올린 차트의 가로축과 같은 길이로 구성
- 축선은 표시하지 않고, 시작점과 끝점의 눈금만 표시
- 평균값의 정확한 위치에 `▼` 표시를 두고, 그 위에 평균값 숫자를 표기

- 가로축: 산출 점수의 산술적으로 가능한 최소값~최대값

#### 2-4-3. 객관식 척도_다중 문항 비교

- 해당 버튼을 누르면 척도 길이가 같은 다른 척도 문항을 체크할 수 있는 창이 뜹니다.

- 세로축: 문항명 (`question_label`)
- 각 문항의 평균값 위치에 점을 표시합니다.
- 점 내부에 평균값 숫자 `xx.x`를 표기합니다.

- 비교 대상 문항들의 그룹별 평균값을 점도표로 중첩해서 표시합니다.
- 가로축: 척도 점수 범위 (`value_count` 기준)
- 세로축: 문항명 (`question_label`)
- 각 그룹의 평균값을 그룹별 색상을 적용한 점으로 표시합니다.
- 점 내부에 평균값 숫자 `xx.x`를 표기합니다.
- 같은 그룹의 점들은 연한 점선으로 연결합니다.

#### 툴팁
- 기본 차트: 표시하지 않습니다.
- 그룹 비교 차트의 점:
  - 1행: 그룹명
  - 2행: 평균 `xx.x점`

#### 범례
- 기본 차트: 표시하지 않습니다.

| 객관식 척도_derived | 바이올린 차트 그라데이션 (붉은색-회색-푸른색) | 동일, 그룹별 색상 차이 없음 |
| 객관식 척도_다중 문항 비교 | 점 색상 통일 | 그룹별 점 색상 베리에이션 (색상 차이) |
| 주관식 숫자 | KDE 곡선 색상 통일 | 그룹별 KDE 곡선 색상 베리에이션 (색상 차이) |
| 주관식 문자 | 텍스트 색상 통일 | 그룹별 텍스트 색상 베리에이션 (색상 차이) |

## 4. 최근 반영 사항

아래 내용은 현재 `dashboard.html` 구현 기준으로 추가 반영된 규칙이다.

### 4-1. 객관식 단일의 기타 응답 텍스트 노출

- 문항 코드북에서 `other_input_expected = Y`인 객관식 단일 문항은, 대응하는 기타 텍스트 열이 응답 데이터셋에 존재할 경우 결과 카드 하단에 `기타 응답` 목록을 함께 표시한다.
- 기타 응답 목록은 차트/데이터 테이블과 분리된 박스 형태로 노출한다.
- 동일한 텍스트가 여러 번 존재하더라도 화면에서는 중복을 제거한 값 목록으로 정리해 보여준다.
- 기타 텍스트 열이 데이터셋에 없거나, 현재 필터 조건에서 유효한 텍스트가 없으면 해당 박스는 표시하지 않는다.

### 4-2. 그룹 비교 범례 배치 및 표시 방식

- 그룹 비교 범례는 차트와 데이터 테이블 사이의 가로 배치가 아니라, 차트 우측의 세로 패널로 배치한다.
- 범례에는 별도 제목을 표시하지 않는다.
- 각 범례 항목에는 체크박스, 색상 스와치, 그룹명만 표시한다.
- 그룹별 `N` 값은 범례 항목에 직접 표시하지 않는다.
- 화면 폭이 좁아 우측 배치가 어려운 경우에는 범례 패널을 차트 아래로 내려 표시할 수 있다.

### 4-3. 그룹 비교 꺾은선/점 정렬 규칙

- 그룹 비교 꺾은선과 점 마커의 세로 위치는 고정된 행 높이 추정값으로 계산하지 않는다.
- 실제 렌더된 각 행의 막대 트랙(`.hbar-track`) 중심 좌표를 읽어, 그 중심을 기준으로 점과 선의 Y축 위치를 계산한다.
- 따라서 보기 수가 많아지더라도 아래 행으로 갈수록 오차가 누적되지 않도록 처리한다.
- 창 크기 변경 또는 결과 재렌더링 이후에도 같은 방식으로 다시 정렬한다.

### 4-4. 최근 규칙의 실제 적용 범위

- `5는 올림(round half up)` 반올림 규칙은 기본 차트 데이터 레이블, 그룹 비교 차트 데이터 레이블, 툴팁, 데이터 테이블에 동일하게 적용한다.
- 코드북 보기 순서 유지와 `0건 보기 숨김` 규칙은 기본 차트, 그룹 비교 차트, 데이터 테이블에 동일하게 적용한다.

### 4-5. 데이터 테이블 표기 문구 보강

- 데이터 테이블에서 기존 `N` 표기는 사용자 친화적인 문구인 `빈도수 (명)`으로 표시한다.
- 그룹 비교 테이블에서도 동일하게 각 그룹의 count 열 제목을 `빈도수 (명)`으로 통일한다.

### 4-6. 질문 원문(question_full) 표시 스타일

- 결과 카드 상단의 질문 원문(`question_full`)은 보조 설명이지만, 기존보다 한 단계 크게 읽히도록 표시한다.
- 현재 구현 기준으로 질문 원문은 본문보다 약간 작은 보조 텍스트가 아니라, 차트 문맥을 빠르게 이해할 수 있을 정도의 크기와 대비를 확보해야 한다.
## Rank Question Rules

The following visualization rules are fixed for every `객관식 순위` question.

- Each rank segment percentage uses the full respondent count for the question as the denominator.
- Group comparison uses each group's full respondent count `n` as the denominator for all rank percentages.
- Responses that exist only in the raw rank answer and do not appear in rank expanded columns are still included in the chart and in the data table.
- Raw-only non-ranked responses are not included in `가중 점수` or `종합 순위`.
- `기타` remains in the rank chart and rank data table, but is always excluded from overall ranking.
- `기타` direct-input text is opened from the table by the shared modal-style `응답 보기` button.
- If a row has no ranked segment but does have raw-only responses, it may render as a single gray bar rather than a multi-segment rank stack.
- When a group is hidden in the legend, that group is removed from both the chart output and the data table.
- Rank group comparison uses a table structure for per-group rank outcomes instead of inline text.
- Rank data tables include a total row; only per-rank percentage and count totals are aggregated, while `가중 점수` and `종합 순위` stay `-`.
- Chart option labels should show the full response label on hover, even when the visible text is truncated.
- Raw-only non-ranked responses in rank questions use a muted red fallback bar and the tooltip format matches other rank segments: option, rank label, percentage, and count.

## Numeric Open Rules

The following visualization rules are fixed for every `주관식 숫자` question.

- Use the question's `raw` column as the source value column.
- Apply filters based on the label dataset, but read numeric values from the numeric dataset when available.
- Treat each response as a continuous numeric value.

### Basic Chart

- Render a histogram as the default chart.
- X-axis uses the observed response range from minimum to maximum.
- Y-axis uses frequency.
- The system proposes a default bin count, and the user can change the histogram bin count directly.
- Show the mean as a bold vertical line with a mean value label.
- Show the minimum and maximum values at the ends of the axis.
- Show the current bin count in the chart footer area.

### Group Comparison

- When a comparison criterion is applied, render the full-response histogram in a light gray tone.
- Overlay each group's response distribution as a colored curved line on top of that histogram.
- X-axis uses the response range by interval.
- Y-axis uses density.

### Data Table

- The default table columns are:
  - `평균`
  - `최소`
  - `Q1 (하위 25%)`
  - `Q2 (중앙값)`
  - `Q3 (상위 25%)`
  - `최대`
  - `응답자 수`
- When group comparison is active, include both `응답자 전체` and per-group rows.

### Tooltip

- Histogram bar tooltip:
  - line 1: range
  - line 2: percentage
  - line 3: respondent count
- Mean marker tooltip:
  - basic chart: mean, respondent count
- Group comparison line tooltip:
  - line 1: group name
  - line 2: range + unit
  - line 3: percentage
  - line 4: respondent count

### Legend

- Basic chart: no legend.
- Group comparison:
  - show group color and label
  - provide checkboxes for group visibility

## Text Open Rules

The following dashboard rule is fixed for every `주관식 문자` question.

- `주관식 문자` is not supported as a chart type in the dashboard.
- Do not render a basic chart for this question type.
- Do not render a group comparison chart for this question type.
- If the question appears in the dashboard, show it as an unsupported visualization state instead of attempting a graph.
- Do not define legend, tooltip, or color rules for `주관식 문자` chart output because chart output is not supported.
